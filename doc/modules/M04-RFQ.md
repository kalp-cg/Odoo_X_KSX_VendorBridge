# M04 ‚Äî RFQ (Request For Quotation)

> Source of truth for RFQ creation and lifecycle. See [09-WORKFLOWS.md](../09-WORKFLOWS.md) ¬ß9.3 and [10-BUSINESS-RULES.md](../10-BUSINESS-RULES.md) ¬ß10.1.

## M04.1 Purpose

- Allow procurement officers to create and manage RFQs.
- Publish RFQs to selected active vendors.
- Track RFQ lifecycle from draft to closed/cancelled.
- Serve as the parent record for quotations, approvals, POs, and invoices.

## M04.2 Scope

**In scope**:
- Create, edit (DRAFT only), publish, close, cancel RFQs.
- Assign vendors (only ACTIVE vendors).
- Add line items (description, quantity, unit, target unit price).
- Attach files (Cloudinary).
- Search and filter RFQs.
- Vendor portal view of assigned RFQs.
- Notifications on publish / close / cancel.
- Audit log of every state change.

**Out of scope**:
- Templates for RFQs (planned v1.1).
- Recurring RFQs.
- Public RFQ listings.
- Multi-currency (single currency in v1).

## M04.6 Workflow

```
DRAFT ‚îÄ‚îÄpublish‚îÄ‚îÄ‚ñ∂ PUBLISHED
PUBLISHED ‚îÄ‚îÄclose‚îÄ‚îÄ‚ñ∂ CLOSED
PUBLISHED ‚îÄ‚îÄcancel‚îÄ‚îÄ‚ñ∂ CANCELLED
```

State machine function: `assertRfqTransition(from, to)`.

| From | To | Allowed |
|------|----|---------|
| DRAFT | PUBLISHED | ‚úÖ |
| PUBLISHED | CLOSED | ‚úÖ |
| PUBLISHED | CANCELLED | ‚úÖ |
| DRAFT | CANCELLED | ‚ùå (use DELETE) |
| DRAFT | CLOSED | ‚ùå |
| CLOSED | * | ‚ùå (terminal) |
| CANCELLED | * | ‚ùå (terminal) |
| * | DRAFT | ‚ùå (no revert) |

## M04.7 Publish invariants

`DRAFT ‚Üí PUBLISHED` requires all of:

- BR-001: ‚â•1 vendor assigned.
- BR-002: ‚â•1 line item.
- BR-003: deadline > now.
- BR-010: all assigned vendors are `ACTIVE`.

If any fails, return 422 with the rule code in `errors[0].code`.

## M04.8 Auto-numbering

`rfqNumber` format: `RFQ-YYYY-NNNN` (per calendar year, monotonically increasing per year).

Generated by a sequence or by a counter row in a `counters` table:

```ts
const year = new Date().getFullYear();
const counter = await tx.counter.upsert({
  where: { name: `RFQ-${year}` },
  create: { name: `RFQ-${year}`, value: 1 },
  update: { value: { increment: 1 } },
});
const rfqNumber = `RFQ-${year}-${String(counter.value).padStart(4, '0')}`;
```

Generated inside the same transaction as the RFQ insert (atomic).

## M04.9 Close and cancel side effects

**Close (PUBLISHED ‚Üí CLOSED)**:
- Any `SUBMITTED` or `SHORTLISTED` quotation is auto-rejected with reason `RFQ_CLOSED`.
- Notifications sent to the affected vendors.
- Audit events written for each auto-rejection.

**Cancel (PUBLISHED ‚Üí CANCELLED)**:
- Same as close, with reason `RFQ_CANCELLED`.
- The `reason` from the request body is stored in the audit metadata.

## M04.11 Attachments

- PDF, JPG, PNG only. Max 10 MB each.
- Cloudinary folder: `vendorbridge/rfq/<rfqId>/<fileId>`.
- Only editable in DRAFT.
- After publish, attachments are read-only (vendor users can view).

## M04.12 Vendor filtering (vendor portal)

- A vendor user sees only RFQs where `RFQVendor.vendorCompanyId = user.vendorCompanyId`.
- AND `status = PUBLISHED` (vendors don't see DRAFT).
- AND `deadline > now` for the "active" list; expired RFQs go to "past" list.
- AND the vendor has not already submitted a quotation (for the "action required" view).

## M04.13 Search and filtering

- `?q=` searches title, description, rfqNumber (trigram).
- `?status=DRAFT|PUBLISHED|CLOSED|CANCELLED`.
- `?deadlineBefore=2026-12-31`.
- `?deadlineAfter=2026-01-01`.
- `?createdById=...`.
- `?vendorId=...` (filter by assigned vendor ‚Äî internal use).
- `?sortBy=deadline|createdAt|title&sortOrder=asc|desc`.

## M04.15 Notifications

- `RFQ_PUBLISHED` ‚Üí all vendor users of assigned vendor companies.
- `RFQ_CLOSED` ‚Üí all vendor users of assigned vendor companies (with reason).
- `RFQ_CANCELLED` ‚Üí same as close.
- (Implicit) `QUOTATION_SUBMITTED` ‚Üí officers (triggered by quotation module).

## M04.16 Edge cases

| Scenario | Behavior |
|----------|----------|
| Publish with 0 vendors | 422 `BR-001` |
| Publish with deadline in past | 422 `BR-003` |
| Publish with a vendor that became INACTIVE | 422 `BR-010` (re-check on publish) |
| Edit a PUBLISHED RFQ's title | 409 `WORKFLOW_INVALID_TRANSITION` |
| Close an RFQ with no quotations | Allowed. No auto-rejections. |
| Cancel an RFQ mid-approval | Auto-rejects the open quotation. Manager sees the rejection reason. |
| Two officers edit the same DRAFT RFQ | Last write wins; both audit-logged. |
| Delete a DRAFT RFQ with attachments | Attachments are deleted from Cloudinary first, then the row. |
| RFQ deadline has passed but status is still PUBLISHED | The deadline is "advisory" for the RFQ status (auto-close not implemented in v1; admin closes manually). Quotations are auto-locked. |
| Vendor portal: vendor not assigned to an RFQ | 404 (the RFQ is not visible to them) |
